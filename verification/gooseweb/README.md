# Gooseweb acceptance contract

This directory is the source-controlled contract for Golden Goose migration browser review. `schemas/` versions the complete nested record shapes, `manifests/` describes product scenarios, `ledger/phase-state.json` is the exact P00-P56 graph and authoritative state/lease history, and `allowlists/` is the complete set of expected browser messages, requests, and WebSocket events. The deterministic validator applies every schema to its document, then enforces cross-record, lifecycle, exact-head, capture, and evidence-file rules that JSON Schema cannot express. It runs in the existing Gooseweb test entrypoint.

Evidence belongs at `tmp/gg/gooseweb-migration/<phase-id>/<sha7>/attempt-<n>/` and must match `schemas/evidence-run.schema.json`. It is intentionally not committed. Each run contains a manifest copy and SHA-256, environment descriptor, screenshots, console/network/WebSocket results, redacted runtime/Tower/store state, check results, applicable metrics, report, and exact-head clearance JSON.

Never retain credentials, cookies, CSRF values, bearer tokens, realtime tickets or query secrets, provider authentication, raw image bytes, secret configuration, or private keys. Redact at capture time; a later cleanup is not an acceptable control. The validator resolves each referenced artifact inside the declared root, requires every file, verifies screenshots are PNGs and the manifest copy hash, scans all text/JSON artifacts, and compares console/HTTP/WebSocket captures to the exact allowlists. Unavailable WebSocket capture is never inferred: it requires a blocked baseline mapping.

Browser acceptance uses one fresh, uniquely named, headless, real-Chromium `agent-browser` session. The descriptor records the exact Chromium binary/version and fresh-profile policy. Headed evidence is rejected, and a browser mode, binary, version, profile, reviewer, lease, manifest, stack, head, tree, or served-head change invalidates clearance. Only the supervisor may own the migration-wide stack lease and lifecycle. Reviewers are read-only and own only browser interaction and evidence.
